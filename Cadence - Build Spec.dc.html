01

Architecture

React Native / Expo app (port the prototype's screens 1:1) │ ▼ HTTPS (user's session JWT) Your backend — Node/Express or serverless (Vercel / Render) │ keeps ALL secret keys server-side. The app never sees them. ├──▶ OpenAI GPT conversation + placement + debrief ├──▶ OpenAI Whisper speech → text (mic audio) ├──▶ ElevenLabs text → native voice (TTS) ├──▶ Azure Pronunciation pronunciation scoring (per-phoneme) ├──▶ Neon Postgres users, progress, decks, logs └──▶ Stripe / Razorpay subscriptions + webhooks

Golden rule: every third-party key lives only on the backend. The app calls your endpoints (e.g. POST /api/conversation); your server attaches the key and proxies. This protects your billing and lets you swap providers without shipping an app update.

04

Auth & payments

🔐 Auth
Screens are designed (Auth + welcome-back toggle). Use Neon + a auth layer: either Auth.js (NextAuth) or Supabase Auth pointed at Neon, or Clerk. Support Google, Apple, and email/password — all three are in the design. Store only a user_id + hashed credentials; issue a JWT the app sends on every API call.
💳 Payments
Checkout screen is designed with Card + UPI/Razorpay toggle and a 7-day-trial summary. Use Stripe (global) or Razorpay (India). Create a subscription with a trial; listen to webhook events to flip users.plan to plus. Never trust the client — the webhook is the source of truth.
